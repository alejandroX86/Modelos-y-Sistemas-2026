📘 Clase 7: Modelos y Sistemas

Escuela N°1 Técnica de Muñiz | 7°4° | Prof. Omar Bazar

🎯 ¿De qué vamos a hablar hoy?

Imagina que miras una foto de un equipo de fútbol, un grupo de WhatsApp, una red de WiFi o incluso tu propio cuerpo. A simple vista parecen cosas distintas, pero todos funcionan con las mismas reglas invisibles.

Hoy aprenderemos a ver el mundo como sistemas. No usaremos fórmulas complicadas como único recurso. Usaremos ejemplos reales, juegos interactivos y observación directa. Al final, entenderás por qué:

  • El todo siempre sorprende más que las partes sueltas
  • Lo que le pasa a uno, tiembla en todos
  • Los sistemas sobreviven porque fluyen, no porque se queden quietos
  • Hay muchos caminos para llegar al mismo resultado
💡 Regla de oro: No necesitas ser un genio de las matemáticas para entender sistemas. Solo necesitas curiosidad y ganas de conectar puntos. En cada sección encontrarás una demostración interactiva para experimentar.

1️⃣ El Todo es Más que la Suma de sus Partes

🧠 Propiedades Emergentes

Hace siglos, los científicos creían que para entender algo debían desarmarlo. Si querías saber cómo funciona un reloj, lo abrías y estudiabas cada rueda. Eso funciona con máquinas simples. Pero ¿qué pasa con cosas vivas o sociales?

🧩 Ejemplo de Lego: Si tiras 100 piezas al suelo, no tienes un castillo. Tienes un desorden. El castillo solo aparece cuando las piezas se organizan y se conectan. La "fortaleza" no está en ninguna pieza sola. Emerge de cómo se unen.

Ludwig von Bertalanffy llamó a esto Propiedades Emergentes. Son cualidades que desaparecen si separas las partes. La fórmula conceptual es:

// Propiedad del sistema completo
Stotal ≠ Σ si
// El todo NO es igual a la suma de las partes
Stotal = Σ si + Einteracciones
// Donde E representa las relaciones entre elementos

En criollo: Si tenés 5 músicos excelentes pero cada uno toca una canción distinta en el mismo escenario, el resultado es ruido. La sinfonía emerge solo cuando se coordinan (interactúan). La música no está en ningún instrumento individual.

2️⃣ Sistemas Conectados (Redes que Respiran)

🕸️ Interdependencia y la Ecuación de Bertalanffy

Un sistema es un conjunto de elementos que se hablan entre sí. No importa si son computadoras, personas, neuronas o semáforos. Lo importante es que se afectan mutuamente.

// Ecuación general de sistema (Bertalanffy)
dQi/dt = fi(Q1, Q2, ..., Qn)
// El cambio de cualquier elemento Qᵢ depende
// de cómo interactúa con TODOS los demás

Traducción: "El cambio de cualquier parte depende de cómo interactúa con todas las demás." Si un router falla, otros trabajan de más. Si un compañero falta, otro cubre su rol. Si sube la temperatura, el ventilador se activa.

🎮 Demostración Interactiva: Red de Nodos Interdependientes

Hacé clic en cualquier nodo (círculo) para "perturbarlo". Observá cómo la perturbación se propaga por toda la red. Luego ajustá el deslizador de acoplamiento para ver cómo cambia la propagación.

3️⃣ Crecimiento y Saturación

📈 Balance de Flujos: Lo que Entra vs Lo que Sale

Todos los sistemas crecen o se achican según un balance simple pero poderoso:

// Ecuación de balance de masa/flujo
dQ/dt = T − P
// T = Transporte (entrada desde el entorno)
// P = Producción/Resistencia (salida o consumo)
// Si T > P → el sistema crece 📈
// Si T < P → el sistema se vacía 📉
// Si T = P → el sistema se mantiene ⚖️
🚿 La Bañera Científica:
Si la canilla tira 5 litros/min y el desagüe drena 3 litros/min, el agua sube a razón de 2 L/min. Si el desagüe drena 7 L/min, el agua baja a 2 L/min. Si ambos son 5, el nivel no cambia. En computación: si llegan más datos de los que se pueden procesar, la memoria se llena y el programa se traba. Eso se llama saturación.

🎮 Demostración Interactiva: Tanque con Flujo y Saturación

Ajustá los deslizadores de Entrada (T) y Salida (P). Observá el nivel del tanque y el indicador de estado. Cuando T supera mucho a P, el tanque se acerca al 100% → ¡saturación!

🛑 Señal de alerta: Cuando "lo que entra" supera mucho a "lo que sale", el sistema grita (se satura, se cae, o colapsa). La solución no es tapar el desagüe, es ajustar el flujo. En redes informáticas esto se llama control de congestión.

4️⃣ Equilibrio Dinámico (Estado Uniforme)

⚖️ Estable pero en Movimiento Constante

Una piedra en el suelo está en equilibrio estático: no se mueve, no cambia, no intercambia nada. Está "muerta" como sistema. Los sistemas vivos y sociales están en equilibrio dinámico (o Estado Uniforme). Parecen quietos, pero en realidad están en constante intercambio.

// Condición de Estado Uniforme (Steady State)
T + P = 0   (balance neto = 0)
// No significa que no haya movimiento
// Significa que lo que entra = lo que sale
// ¡Hay flujo constante pero el nivel no cambia!
🏫 La Escuela N°1 Técnica de Muñiz:
Cada año se van los de 7°, entran los de 1°. Cambian profes, se pintan paredes, se renuevan cuadernos. Sin embargo, "La Escuela" sigue siendo la misma institución. ¿Por qué? Porque lo que sale se reemplaza por lo que entra. La estructura se mantiene gracias al flujo, no a la quietud.

🎮 Demostración Interactiva: Equilibrio con Partículas Visibles

Ajustá los deslizadores para que T = P (por ejemplo, ambos en 50). Verás partículas azules entrando y saliendo constantemente, pero el nivel se mantiene estable. Eso es equilibrio dinámico: movimiento perpetuo, resultado constante.

5️⃣ Equifinalidad

🎯 Múltiples Caminos, Mismo Destino

Equifinalidad significa que un sistema abierto puede llegar al mismo estado final partiendo desde lugares distintos y siguiendo rutas diferentes. Es lo opuesto al determinismo rígido de las máquinas simples.

// Principio de Equifinalidad (Bertalanffy)
Estadofinal = F(condicionesiniciales, rutaelegida)
// Diferentes combinaciones (condiciones + ruta)
// pueden producir exactamente el mismo resultado
// No hay UN solo camino correcto
🍳 Tres recetas, un mismo pastel:
- Persona A: Usa harina integral, masa madre, horno lento.
- Persona B: Usa harina blanca, levadura rápida, horno eléctrico.
- Persona C: Usa premezcla, microondas, 3 minutos.
Resultado: Los tres obtienen un pastel comestible. Diferentes materiales, distintos tiempos, mismo objetivo cumplido.

🎮 Demostración Interactiva: Tres Caminos al Mismo Nivel

Tenés tres configuraciones predefinidas (A, B, C) con distintos valores de Entrada (T) y Salida (P). Todas llevan al mismo nivel de equilibrio (~50%). Probá cada una y observá que, aunque los valores individuales son distintos, la diferencia neta es la misma.

Nivel actual: 50% 🎯 Mismo destino
🌍 Lección: No existe un único camino "correcto". La inteligencia sistémica es saber adaptarse al contexto y usar los recursos disponibles para alcanzar el equilibrio o la meta. Diferentes lenguajes de programación → misma app. Diferentes rutas de colectivo → misma escuela. Diferentes técnicas de estudio → mismo aprobado.

🎯 5 Actividades Grupales (5 grupos × 5 estudiantes)

Cada actividad dura ~15-20 min. Materiales: papel, lápiz, cronómetro del celular, objetos simples del aula.

Grupo 1: "La Torre Sin Palabras"

Concepto: Propiedades Emergentes
Roles: 2 constructores (sin hablar), 1 observador, 1 registrador, 1 cronometrador.
Paso: Con 20 hojas y cinta, armen la torre más alta en 5 min. Prohibido hablar.
Cierre: ¿Qué estrategia surgió sin palabras? ¿La torre es más que hojas sueltas?

Grupo 2: "Teléfono Sistémico"

Concepto: Interdependencia
Roles: 1 emisor, 3 intermediarios, 1 receptor final, 1 observador externo.
Paso: Pasa un mensaje complejo. En la vuelta 2, un intermediario cambia una palabra.
Cierre: ¿Cómo un cambio pequeño alteró el resultado? ¿Quién fue el "nodo crítico"?

Grupo 3: "El Balde de Datos"

Concepto: Crecimiento y Saturación
Roles: 1 "entrada" (echa agua con vaso), 1 "salida" (saca con esponja), 1 "medidor", 1 "cronometrador", 1 "registrador".
Paso: Simulen 3 escenarios: entra > sale, entra < sale, entra = sale.
Cierre: ¿En qué momento se "saturó"? ¿Cómo lo evitarían en un servidor real?

Grupo 4: "La Escuela que Fluye"

Concepto: Equilibrio Dinámico
Roles: 2 "alumnos nuevos", 2 "graduados", 1 "director/regulador".
Paso: Entran y salen cada 30 seg. El director debe mantener 3 personas "activas" en el centro.
Cierre: ¿El sistema se mantuvo estable? ¿Qué pasaba si se detenía un flujo?

Grupo 5: "Puentes Diferentes"

Concepto: Equifinalidad
Roles: 1 arquitecto, 2 ingenieros, 1 probador, 1 registrador.
Paso: Reciben materiales distintos (clips, papeles, regla, elástico). Deben hacer un puente que sostenga un libro.
Cierre: ¿Lograron lo mismo con recursos distintos? ¿Qué estrategia fue más eficiente?

📌 Instrucción para el docente: Rote entre grupos. No dé respuestas, haga preguntas: "¿Qué pasó cuando X cambió?", "¿Cómo se ajustaron?", "¿Qué les recuerda esto a una red WiFi?". Al final, 1 representante de cada grupo comparte 1 conclusión en 1 min.

👨‍🏫 Guía Rápida para el Docente

📋 Secuencia Sugerida (80 minutos)

  1. Inicio (10 min): Presentar la idea "El todo > suma de partes" con el ejemplo del Lego o equipo de fútbol. Usar la demo del Autómata Celular.
  2. Explicación (20 min): Recorrer las 5 secciones con las demostraciones interactivas. Evite derivar fórmulas directamente; deje que los estudiantes jueguen con cada demo y luego pregunte: "¿Qué observaron?"
  3. Actividades Grupales (25 min): Divida en 5 grupos de 5. Asigne 1 actividad por grupo. Circule, observe, anote frases clave.
  4. Puesta en común (10 min): 1 min por grupo. Vincule cada experiencia con el concepto teórico y la fórmula correspondiente.
  5. Simulador + Cierre (15 min): Proyecte el simulador integrado. Dejen que manipulen. Cierre con la idea de equifinalidad.

🔍 Evaluación Formativa (Sin exámenes)

  • ✅ Participación en roles grupales
  • ✅ Capacidad de conectar el juego con el concepto
  • ✅ Uso correcto de vocabulario sistémico (flujo, equilibrio, red, saturación, emergencia)
  • ✅ Reflexión escrita breve (3 líneas): "¿Cómo aplicaría esto a mi celular/comunidad?"
🌟 Frase final para la clase:
"Hoy no aprendimos fórmulas como recetas de cocina. Aprendimos a ver conexiones. Entendimos que nada existe aislado, que crecer requiere balance, que hay muchos caminos válidos, y que lo más importante de un sistema nunca está en una pieza sola, sino en cómo bailan juntas."
